### 技术突破

- **SafetyALFRED**：评估 MLLM 在物理环境中的主动安全规避能力。核心发现：模型在 QA 设定下能识别危险，但在 embodied 场景中主动缓解率很低——"识别危险"和"主动行动规避"之间存在显著差距。
- **MAGEO（Multi-Agent GEO）**：将生成引擎优化从单次任务变成策略学习问题。多 Agent 协作（规划/编辑/评估），将有效编辑模式蒸馏为可复用技能。发布 MSME-GEO-Bench 多场景 benchmark。
- **TSAG（Time Series Augmented Generation）**：金融场景 LLM Agent 评估框架，100 个金融问题，测试工具选择准确性、忠实度和幻觉率。GPT-4o、Llama 3、Qwen2 近满分工具使用准确率。
- **DT2IT-MRM**：多模态 Reward Modeling 新 SOTA，整合去偏偏好构建 + T2I 偏好数据重表述 + 迭代训练。在 VL-RewardBench 等三个主要 benchmark 刷新记录。

## 🎯 老大论文解读（推荐 2 篇）

### ① SafetyALFRED（强烈推荐）
**《Evaluating Safety-Conscious Planning of Multimodal Large Language Models》**
arXiv:2604.19638 | 11 个 SOTA 模型 | SafetyALFRED benchmark

**一句话核心**：现在的 MLLM（Qwen/Gemma/Gemini 全系列）在 QA 设定下能识别危险，但在真实 embodied 物理环境中主动规避危险的成功率极低。静态 QA 评估完全无法捕捉真实的物理安全风险。

**对老大的价值**：
你想做"个性化学习 Agent"，核心问题之一是 Agent 如何在真实环境（不只是聊天框）里主动做事。SafetyALFRED 揭示了一个所有 Agent 开发者的共同盲点：**模型知道危险在哪 ≠ 模型会主动去规避危险**。你用 Claude Code 写代码时，如果 Agent 给你一个错误方案，它能识别出来吗？它会主动修正吗？这篇论文用实验证明：大多数模型"知道但不做"。这对 Agent 设计中的"自我纠正机制"设计非常重要。

---

### ② MAGEO — Multi-Agent 生成引擎优化
**《From Experience to Skill: Multi-Agent Generative Engine Optimization via Reusable Strategy Learning》**
arXiv:2604.19516 | MSME-GEO-Bench | Planning/Editing/Fidelity 三层 Agent

**一句话核心**：将 SEO（搜索引擎优化）式的"单次内容优化"变成多 Agent 协作的策略学习问题——规划 Agent 生成编辑策略、编辑 Agent 执行修改、评估 Agent 判断质量，有效策略被蒸馏成可复用的 engine-specific 技能。

**对老大的价值**：
这篇论文对你的意义不在于 GEO 本身，而在于它展示了一个**可扩展的多 Agent 协作架构**：多个 Agent 有不同角色（规划/执行/评估），通过协作完成任务，有效结果被积累成可复用资产。这和你想做的"个性化学习 Agent"在架构上有相似之处——学习规划 Agent 给你规划路径、执行 Agent 帮你拆解任务、评估 Agent 判断你掌握的程度。MAGEO 证明这类架构是可行的，而且比单 Agent 效果更好。
