论文一：Synthetic Computers at Scale（2604.28181）★★★ 首选
━━━━━━━━━━━━━━━━━━━━━━━━

【核心贡献】
Meta（Facebook Research）构建了 1000 个"合成计算机"——模拟真实用户工作环境的虚拟电脑，每个包含完整的文件系统（文件夹层级、文档/表格/PPT 等富内容 artifact）。

两个 Agent 协作：一个是"用户"（提出一个月量的 productivity objectives），另一个扮演该用户在电脑上执行任务。平均每次模拟 8 小时、2000+ 轮次，生成高质量的 long-horizon 经验轨迹。最后用这些轨迹微调 Agent，显著提升在内域和外域的 productivity 任务表现。

【为什么重要】

这是 AI 训练数据生成范式的第三代：
  - 第一代：人工标注（SFT）
  - 第二代：RLHF，人类偏好标注
  - 第三代：**让 Agent 在模拟真实环境中扮演用户，自己生成训练数据**

不需要人工标注，不需要人类真实数据。Agent 在"模拟人生"里学会真实的工作技能。

【对老大学习 Agent 的直接价值】

① 理解"环境模拟"在 Agent 训练中的核心地位：
  - Agent 的能力不仅来自模型，还来自它所处的"世界"
  - 1000 个合成电脑 = 1000 个不同的"世界观"，让 Agent 泛化到新环境

② "合成用户 + 合成世界"的思路可以迁移到你的个性化学习 Agent：
  - 模拟学生的学习环境（不同学科、不同难度梯度）
  - Agent 在模拟环境中练习"教学"，生成高质量教学轨迹
  - 用这些轨迹训练 Agent 的教学能力

③ 工程直觉：规模化是核心杠杆
  - 1000 个合成环境 × 8 小时 × 2000+ 轮次 = 百万级训练信号
  - 质量 × 数量 = 突破性能瓶颈

【难度】中等。概念直观（模拟人生训练 AI），工程规模惊人，适合作为"合成数据"概念的入门案例。

论文二：Intern-Atlas — 100万论文构建方法演进图（2604.28158）★★★ 备选
━━━━━━━━━━━━━━━━━━━━━━━━

【核心贡献】
从 1,030,314 篇 AI 论文中自动提取方法级实体（如 SFT、RLHF、DPO），并构建方法间的血缘关系网络——标注"这个方法从哪里来、解决什么问题、为什么出现"。

关键数字：9,410,201 条有语义类型的边，可以查询"方法家族树"。

【为什么值得关注】

① 科学的本质是方法的累积，但 AI 领域一直没有结构化的知识基础设施
② 这篇论文提供了一个**可查询的方法演进数据库**
③ 用途：
  - 研究新方法时，直接追溯祖先和后代
  - 比如研究 DPO：SFT 有什么问题 → RLHF 怎么解决 → DPO 又怎么改进 RLHF
  - 比读散落的论文高效 10 倍

【对老大学习 AI 的工具价值】

你在学 Agent 框架时，一定会遇到很多方法名字（SFT、RLHF、DPO、ReAct、CoT……）。Intern-Atlas 让你：
  - 看到每个方法的"家谱"
  - 理解"为什么这个方法会出现"
  - 建立 AI 领域方法的系统认知，而不是碎片化记忆

【难度】中等。工程规模大，概念容易理解。可以当作"AI 领域的方法论维基百科"来用。
