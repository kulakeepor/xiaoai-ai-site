## 四、老大论文解读（arXiv 本周重点）

### 📄 论文1：物理学家监督 AI 编程 Agent — CLAX-PT 案例研究
**arXiv: [2605.30353](https://arxiv.org/abs/2605.30353)** | ICML 2026 AI for Science Workshop

**背景**：一位物理学家用 Claude Code（Sonnet + Opus）历时 12 天、57 个 session 开发 CLAX-PT（JAX 可微一阶微扰理论模块）。

**核心发现（15 次监督事件分类）**：
- Agent 自主解决 10 个问题（通过 oracle 测试迭代）
- 物理学家介入解决 2 个（领域知识）
- **3 个无法解决**——这3 个都共享一个共同问题：Agent 把"症状减轻"当作"根因解决"，在错误代码架构内调参数

**关键教训**：
1. Oracle 测试无法捕获的3 类失效：架构选择错误、越过架构重新评估、无物理意义的数值补丁
2. **监督设计比模型能力更重要**：三种关键监督实践——多参数点测试、共享 changelog 暴露停滞探索、明确禁止无物理意义的数值补丁
3. 未来方向：Agent 应能提出架构替代方案，而非在给定结构内优化；应区分"预测充分性"与"解释正确性"

**对老大的价值**：这篇是"AI for Science"实操必读。对于你的个性化学习 Agent，核心启示是：**测试不能只验证结果，要验证过程**——学习 Agent 也需要类似的"过程监督"，防止在错误的学习策略内持续优化而不自知。

### 📄 论文 2：多组件 LLM Agent 的组合不相干问题
**arXiv: [2605.30335](https://arxiv.org/abs/2605.30335)** | ICML 2026 CTB / AgenticUQ / FAGEN

**核心问题**：多组件 LLM Agent 从各自只看到部分问题的组件中组合概率陈述，即使每个组件本地相干，组合后也可能违反基本概率公理。

**形式化方法**：
- 提出 **compositional residual ε\***：衡量组合后引用与联合相干多面体的 L2 距离，可在运行时从系统输出和跨组件耦合约束计算
- **乘积-结构二分法**：刻画何时本地相干足够、何时不充足
- **Rayleigh 商数预测**：在四类关系中的三类，预测与观测残差误差在 7% 以内

**修复方案**：
- **分层 Boyle-Dykstra 投影**：确定性修复组合不相干
- **随时有效 e-process**：顺序相干监控

**实验数据**：
- 1,876 个集成 cliques 中，ε\* > 0 的占 33-94%（取决于关系类别）
- 1,770 个已解决赌注中，每次赌注 regret +0.115 nats（比例分配规则下）

**三种直观的 LLM 侧缓解均失效或退化**：检索、分区感知 prompt、聚合器 LLM。

**对老大的价值**：这篇揭示了**多 Agent 系统的概率不相干风险**。对于多 Agent 协调架构（规划 Agent / 执行 Agent / 评估 Agent），组合输出的相干性是一个被忽视但关键的问题——三个 Agent 各自正确，组合后可能产生不合逻辑的结论。值得在架构设计时纳入风险评估。
