## 📰 今日 AI/产品/技术/社区动态（2026-04-26）

### 大厂动态
- **OpenAI 发布 Workspace Agents**：面向企业的 Agent 产品，定位为 Custom GPTs 的企业继承者。可连接 Slack、Salesforce、Google Drive、Notion 等主流办公应用，基于 Codex 云端执行引擎，支持持久化记忆、定时调度、跨应用协作。5 月 6 日起开始按用量收费。

### 技术突破
- **Stanford 研究：单 Agent 在同等思考预算下超越多 Agent 系统**：当控制思考 Token 数量时，单 Agent 在复杂多跳推理任务上与多 Agent 打平甚至更好。多出来的 Agent 间通信开销是"swarm tax"。多 Agent 的真正优势只在单一 Agent 上下文已严重损坏或过长时出现。
- **LLM 行为漂移监控**：企业级 LLM 上线后，同一 prompt 周一和周五输出可能不同。VentureBeat 提出"AI Evaluation Stack"两层方案：Layer 1 确定性断言（正则/结构校验）+ Layer 2 模型评判（LLM-as-Judge），离线 golden dataset + 在线遥测结合。
- **VAKRA Agent Benchmark**：IBM Research 发布，评估 Agent 在真实任务中的推理、工具使用和失败模式。

### 企业/生态

### 必读 #1：DiffMAS [2604.21794] — 多Agent隐通信，端到端联合优化

**背景与问题**
现有 Multi-Agent LLM 系统，大多数工作关注 Agent 角色和编排，但 Agent 间如何"交流"是固定的——靠文本/消息传递。这相当于让两个大脑之间只能靠写信来协作，而不是共享神经活动。

**核心贡献**
DiffMAS 将隐含通信（latent communication）引入多 Agent 系统。Agent 通过共享 Key-Value cache 而非文本交换信息，在内部表征层面直接对齐。训练时用参数高效的监督学习在多 Agent 轨迹上联合优化，让 Agent 学会"编码什么"和"如何理解"。

**关键数字**
- AIME24（数学竞赛）：26.7%（DiffMAS） vs 对比基线全面提升
- GPQA-Diamond（科学推理）：20.2% 提升
- 数学/科学QA/代码/常识推理 benchmark 一致性提升

**为什么重要**
这是 COLM 2026 投稿级别工作，代表了多 Agent 研究的前沿方向。隐含通信的好处是避免信息在"说→听→理解"链条上的损失，特别适合需要多 Agent 联合推理的复杂任务。

**对资源库学习的启发**
你想做的个性化学习 Agent，如果需要多个 Agent 协作（规划 Agent + 执行 Agent + 评估 Agent），DiffMAS 证明了隐含通信比文本协议更高效。你的场景不需要从零训练，但这个思路值得在系统设计时考虑——让不同 Agent 共享一个"context cache"，而不是靠 prompt 来传递信息。

### 选读 #2：[2604.21910] — Agentic AI 让科学家用自然语言驱动科学工作流自动化

**背景与问题**
科学家做研究最痛苦的环节之一：把研究问题转化为可执行的工作流（DAG），这需要同时懂科学和工程。现有的科学工作流系统（如 Hyperflow）只解决执行层（调度/容错），不解决"语义翻译"层。

**核心贡献**
三层架构：
1. **语义层**：LLM 将自然语言研究问题解析为结构化意图
2. **确定层**：验证后生成可复现的 DAG（Workflow）
3. **知识层**：领域专家编写"Skills"（markdown 文档），编码术语映射、参数约束、优化策略

关键发现：Skills 让意图匹配精度从 44% 提升到 83%；数据转移减少 92%；单次查询成本 < $0.001。

**为什么重要**
对"AI + 垂直领域"的产品思路有直接参考价值。这里的"知识层 Skills"和你想做的"学习路径规划"有相似性——都是把领域知识结构化，让 AI 能理解并执行。Skills 的设计模式（术语映射+参数约束+优化策略）可以借鉴到你对学习内容的设计中。
