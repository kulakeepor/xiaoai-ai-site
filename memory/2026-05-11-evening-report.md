## 三、arXiv 论文跟踪（本周重点，5月8日-11日）

### ★ Paper 1：TraceFix — TLA+ 修复多 Agent 协调协议（cs.AI，★★★★★ 强烈推荐）

**arXiv:2605.07935 | James Wang et al. | cs.AI/cs.MA | 5月8日**

**为什么重要**：这是第一篇用 TLA+ model checker 实际修复多 Agent 协调协议的论文。48 个任务全部 TLC 验证通过，62.5% 第一次就通过，最多 4 次迭代。Runtime 验证：TLC verified protocols 把 deadlock/livelock 从 31.1% 降到 14.1%。

**核心发现**：

**TraceFix Pipeline**：
1. Agent 从任务描述合成协议拓扑（structured IR）
2. 生成 PlusCal 协调逻辑
3. 用 TLA+ model checker (TLC) 的 counterexamples 迭代修复
4. 验证通过后编译成 per-agent system prompts
5. Runtime monitor 拒绝 out-of-topology 的协调操作

**关键创新**：
- 把 formal verification 从"设计后检查"变成"设计内嵌"
- Counterexample-driven repair：不需要理解为什么错，让 model checker 告诉你哪里错
- Verified protocols 在 model capability 降低时，runtime degradation 是 prompt-only 基线的一半

**效果**：
- 48 tasks × 16 scenarios，100% TLC 验证完成
- 62.5% first-attempt pass，max 4 iterations
- Runtime: 89.4% avg task completion (81.5% full completion)
- DL/LL 从 31.1% → 14.1%（fault injection 下差距最大）

**为什么适合老大**：这篇论文是"让 AI 自己修 AI" 的好案例——不是让 AI 自己修代码，而是让 formal verification 工具修 AI 的协调协议。你学 Agent 框架代码时，最大的问题之一是"多个 Agent 怎么不打架"，TraceFix 给了一个系统性的工程路径。

### ★ Paper 2：Rubric-Grounded RL — 结构化评判奖励（cs.AI，★★★★ 推荐）

**arXiv:2605.08061 | Manish Bhattarai et al. | cs.AI | 5月8日**

**为什么重要**：GRPO（Group Relative Policy Optimization）是当前 LLM 推理训练的主流方法（Qwen-Math、DeepSeek-Math 都用），但 GRPO 需要正负样本对比，在稀疏奖励下负样本信号不够强。这篇论文提出了 Rubric-Grounded RL，用结构化 rubric 替代单一 score，让 reward 信号更细粒度。

**核心发现**：

**GRPO 的 limitation**：GRPO 需要正负样本对比，但在只有 binary reward（对/错）时，负样本的"失败程度"没有 gradation，采样覆盖不了有意义的 reward 信号。

**Rubric-Grounded RL**：
- 把 reward 分解成加权的、可验证的标准
- LLM judge 对每个 criterion 打分，而不是单一 holistic score
- 用 OSTI 衍生的 10 万篇科学文档构建 rubrics
- GRPO 训练 Llama-3.1-8B-Instruct

**效果**：
- Held-out rubric evaluation: 71.7% normalized reward
- 跨 corpus 泛化：GSM8K、MATH、GPQA Main、Diamond 都有提升
- 证据：结构化、文档锚定的 reward 可以诱导跨 corpus 的推理行为

**为什么适合老大**：GRPO 是你学 RL 训练时绕不过的方法。这篇论文告诉你 GRPO 的 reward design 可以更细——不是 binary，而是 rubric 化的多维评分。这对学习伙伴系统的训练模块设计有直接参考价值。

## 四、老大论文推荐

### 今晚推荐阅读：TraceFix（2605.07935）

**推荐理由：**

1. **第一次验证"formal verification 修 AI 协调"可行**：48 tasks，100% TLC verification，62.5% first-attempt。数字本身说明方法行得通。
2. **和你的学习直接相关**：你在学 Agent 框架代码，多 Agent 怎么协调不打架是核心工程问题。TraceFix 给的答案是：用 model checker 强制验证协调拓扑，而不是靠 prompt engineering 试错。
3. **Runtime 鲁棒性有数据**：TLC-verified protocols 在 model capability 降低时，degradation 速率只有 prompt-only 基线的一半。这是工程上可以直接用的结论。
4. **Counterexample-driven repair 的哲学**：不需要理解为什么错，让工具告诉你哪里错。这是一种新的 AI 开发范式——不是 prompt more，而是 verify more。

**次选**：Rubric-Grounded RL（2605.08061）——如果你对 RL 训练有兴趣，GRPO 的 reward 设计优化是个很实际的方向。
