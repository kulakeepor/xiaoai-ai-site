### 重点论文 1：Senses Wide Shut — 表征-行动差距
**arXiv:2605.13737** | 2026-05-13 提交

**一句话：** 多模态 LLM 能「看到」矛盾，但不会「拒绝」错误答案——瓶颈在翻译层，不在感知层。

**背景：**
当前 Omnimodal（大语言模型同时处理视频、音频、文本）被定位为「感知锚定的 Agent」。但这个研究测试了一个最基本的能力：当文本前提与模型自身感知输入矛盾时，模型能否 catch 这个矛盾？

**方法：**
研究者构建了 IMAVB 基准——500 个电影片段，2×2 设计（目标模态 × 前提条件：标准/误导）。测试了 8 个开源模型 + Gemini 3.1 Pro。

**核心发现：**
1. **存在表征-行动 gap**：隐藏状态（hidden states）能够可靠编码「 premise 与感知不匹配」的信号，但模型几乎从不在输出中拒绝错误信息。
2. **两种失败模式：**
   - **欠拒绝（Under-rejection）**：对误导性问题直接回答，忽略矛盾前提
   - **过拒绝（Over-rejection）**：确实更多拒绝，但也拒绝标准问题，牺牲正常理解力
3. **模态不对称**：音频接地显著弱于视觉接地
4. **7 种 prompt 变体都无法修复**：说明不是 prompt engineering 问题

**作者的诊断干预：**
提出了 PGLA（Probe-Guided Logit Adjustment）——将隐藏状态中的不匹配信号重新注入解码过程，一致性改善了拒绝行为。

**关键结论：**
> "The bottleneck for omnimodal grounding lies in **translation**, not perception."

**我的判断：** 这篇论文对做 AI 学习助手有直接参考价值。你在做 AI 学习助手 MVP 时，如果涉及多模态输入（比如用户上传图片/视频），需要意识到：模型能「看到」问题，但不等于会「行动」（拒绝/纠正）。需要在产品层额外加一层验证逻辑，而不能假设模型能自主拒绝错误前提。

### 重点论文 2：Cognifold — 主动记忆系统
**arXiv:2605.13438** | 2026-05-13 提交

**一句话：** 受人脑启发的「始终开启」主动记忆系统，让 AI Agent 能够像人一样从经验中主动构建认知结构，而不只是被动检索。

**背景：**
现有 Agent 记忆主要是「被动检索式」的——存储嵌入 + 检索相关上下文 + 逐步累加状态。但这种方式缺少主动组织能力，无法将碎片经验整合为持久认知结构。

**核心设计：**
Cognifold 基于互补学习系统理论（CLS），将传统的双层结构（海马体 → 新皮层）扩展为**三层**：

```
事件流 → 前额叶意图层（新增）
         ↕ 主动决策和意图控制
         新皮层（高层次认知结构）
         ↕
         海马体（快速学习片段记忆）
```

**五大核心机制：**
1. **主动组装（Self-Assembly）**：认知结构在事件流下主动形成，不用等到检索时才组装
2. **语义合并（Semantic Merging）**：相似的结构会自动合并，减少冗余
3. **衰减（Decay）**：过时的结构自然衰减，不累积垃圾
4. **联想召回（Associative Recall）**：通过联想而非精确匹配来召回记忆
5. **意图浮现（Intent Emergence）**：当概念簇密度超过阈值，意图自动浮现

**评估：**
在 CogEval-Bench（结构形成评估）和 7 个跨 5 个认知领域的基准测试中均表现稳健。证明结构化记忆不会牺牲传统记忆任务性能。

**我的判断：** 这篇论文对「AI 学习助手 MVP」的核心价值最高——它解决的是「记忆」问题，而你的学习助手定位恰好是「动机层」，需要记住用户的：学习历史、进度、偏好、薄弱点。如果能借鉴 Cognifold 的三层架构和主动记忆机制来做设计，会比目前任何 RAG 方案都更接近「真正懂你的学习伴侣」。
