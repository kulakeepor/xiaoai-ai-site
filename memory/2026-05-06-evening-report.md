论文一：OpenSeeker-v2（2605.04036）★★★
━━━━━━

【核心贡献】
来自学术团队（Rui Ye 等）的搜索 Agent 研究。关键发现：**仅用 10.6k SFT 数据，通过三大数据合成改造（知识图谱规模扩展、工具集扩展、低步数严格过滤），在 ReAct 范式下 30B 模型达到 SOTA**，超越通义深度研究（Tongyi DeepResearch，后者用了 heavy CPT+SFT+RL 管线）。

4 个基准结果：
- BrowseComp: 46.0% vs 43.4%
- BrowseComp-ZH: 58.1% vs 46.7%
- Humanity's Last Exam: 34.6% vs 32.9%
- xbench: 78.0% vs 75.0%

这是**首个在同等规模和范式下由纯学术团队开发的 SOTA 搜索 Agent**。

【为什么值得关注】

① 数据质量 > 数据数量：
10.6k 数据胜过大工厂流水线——"知识图谱大 + 工具集广 + 严格低步数过滤"三大改造是关键。这与"大模型 + 大量数据"的直觉相反。

② 搜索 Agent 的民主化：
证明搜索 Agent 不再是大厂专属；普通学术团队用 SFT 就能做出 SOTA。这对学习 Agent 的开发模式有直接参考价值。

③ 对老大学习 Agent 的直接价值：
你的学习资源库本质上是"搜索 + 整理"任务。OpenSeeker-v2 的数据合成策略——大知识图谱 + 多工具 + 严格过滤——可以直接迁移到"学习轨迹"的质量控制上：用高质量小数据集而非低质量大数据集。

【难度】中等。方法简洁，结论反直觉，适合作为"数据策略设计手册"参考。

论文二：TraceLift（2605.03862）★★★
━━━━━━

【核心贡献】
来自 Junhao Su 等的研究，提出 TraceLift 框架——训练推理 Planner 时，不仅奖励最终答案正确，还要奖励**推理追踪本身的质量**。核心机制：Frozen Executor 将 Planner 的推理追踪转化为最终 artifact，然后用"推理追踪在同一个 Executor 上的提升度"作为奖励因子。提出 TRACELIFT-GROUPS——同题多轨迹数据集（含高质量参考轨迹 + 多种缺陷轨迹）。

【为什么值得关注】

① 解决了 RL 推理的核心问题：
当前 RL 用 final-answer correctness 作为 reward，存在"正确答案但错误推理路径"的问题。这会强化错误推理、奖励捷径、传播有缺陷的中间状态。

② 推理追踪的可学习性：
TraceLift 证明推理质量可以直接学习，而不只是通过答案正确性的隐式反馈。这对构建"能改进自己推理能力"的学习 Agent 有直接意义。

③ 对老大学习 Agent 的直接价值：
你的学习 Agent 每天帮老大拆解知识。如果推理链质量低（比如跳步、"显然可得"式省略），学习效果会打折扣。TraceLift 的 executor-grounded reward 可以演化为"学习路径质量评估器"——不仅看用户是否理解，还要看推理路径是否扎实。

【两篇论文的关联】

OpenSeeker-v2 告诉我们：**数据质量比数量更重要**，TraceLift 告诉我们：**推理链质量比答案正确更重要**。

结合起来看老大的学习 Agent：
→ 不需要海量学习资料（OpenSeeker-v2 的小数据策略）
→ 需要高质量的学习推理路径（TraceLift 的追踪质量评估）

这是构建高效学习 Agent 的两个核心原则：精资料 + 稳路径。
