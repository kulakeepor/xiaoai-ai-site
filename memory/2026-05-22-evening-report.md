## 三、老大晚间论文解读（精选）

### Paper 1：DeepWeb-Bench — 深度研究的核心瓶颈不是"搜不到"而是"推不出"
**arXiv:2605.21482 | Xie, Shi, Shen et al. | 北大团队 2026-05-20**

**为什么选这篇给老大细读：**

这是本周最重磅的新提交，它回答了一个关键问题：**当前最强的大模型做"深度研究"时，到底差在哪里？** 反直觉的发现将直接影响学习 Agent 的设计方向。

**核心发现：**

① **检索不是瓶颈** — 现有 benchmark 以为"能否搜到正确信息"是关键，但 DeepWeb-Bench 发现检索失败仅占 12-14% 的错误。真正的问题在于**推导（derivation）**和**校准（calibration）**，两者合计占 70%+ 的失败。

② **强模型和弱模型的失败模式有本质区别**：
- 强模型（GPT-4、Claude 等）：错误主要是"推导不完整"——能找到足够证据，但多步推导时漏掉关键步骤
- 弱模型：错误主要是"精确性幻觉"——给出看起来精确但实际错误的答案

③ **跨模型一致性很低**（Spearman rho=0.61），同一案例上不同模型得分差异可达 18.8 个百分点。说明当前没有全能选手，各模型在深度研究上各有专长。

④ **失败模式因难度级别而异**：简单任务主要失败于检索，中等难度主要失败于校准，困难任务主要失败于推导。

**对老大 AI 学习 Agent 的直接价值：**

1. **评估重心要从"找没找到信息"转向"推导链完不完整"** — 学习 Agent 在给出答案时，应该让用户看到多步推导过程，而不是只给结论。

2. **强模型失败 = 推导不完整** → 学习 Agent 需要"推导追踪"能力，记录每一步推理依赖了哪些证据，而非黑箱式输出。

3. **自适应难度设计** — 不同难度任务的主要失败模式不同，学习 Agent 的自我修正机制也应该差异化：简单问题→检查检索准确性；复杂问题→检查推导链完整性。

4. **多模型协作架构** — 跨模型一致性仅 0.61，单一模型做深度研究有局限，可以考虑让不同模型负责不同研究阶段（检索型 + 推理型 + 校准型）。

**一句话总结**：DeepWeb-Bench 揭示了深度研究的核心瓶颈不是"搜不到"，而是"推不出" — 这意味着学习 Agent 评估学习效果的核心指标，应该是推导链的完整性，而非信息检索的覆盖率。

### Paper 2：CopT — 推理的顺序革命，从"想好再答"到"先答再看需不需要想"
**arXiv:2605.20075 | Shi et al. | cs.CL 2026-05-19**

**为什么选这篇：**

这是推理范式的一次重要翻转，解决的是 Chain-of-Thought 的核心浪费问题：**模型其实可以在扩展思考之前就识别出正确答案，但仍被迫执行完整的 CoT 流程**。对于需要根据问题复杂度动态调整学习深度的学习 Agent，这个思路可以直接迁移。

**核心方法：**

- **Step 1**：先生成草稿答案（draft answer）
- **Step 2**：用连续 embedding 对比验证器判断草稿可信度（对比离散 token 输入 vs. 连续 embedding 输入下模型对答案的支持度）
- **Step 3**：如果可信度不够，再触发深度思考

数学上：验证器的期望值等价于"未解决潜在状态与答案 token 之间的互信息" — 它捕捉的是答案相关的不确定性，而非任意不确定性。

**关键数字**：数学/代码/Agent 推理任务，准确率提升最高 23%，token 使用量降低最高 57%（同等准确率下）。无需额外训练。

**对老大学习 Agent 的价值：**

学习 Agent 面对不同复杂度的学习任务，可以用 CopT 思路自适应决定：
- **简单事实性问题** → 直接给答案，不触发深度推理
- **需要多步理解的概念** → 先生成草稿，再决定是否需要详细推导
- **复杂研究任务** → 完整多步推理 + 推导链可视化

这比"所有问题都用 CoT"节省 token 和时间，同时保持甚至提升答案质量。

**一句话总结**：CopT 把推理从"暴力枚举所有思考路径"变成"先猜再验证" — 对学习 Agent，这意味着根据问题复杂度自适应分配推理预算，是提升效率的关键。
