## 📚 本周 arXiv 重点论文

### 🔬 推荐解读的论文

**① Generalization in LLM Problem Solving: The Case of the Shortest Path**
- ID: arXiv:2604.15306
- 链接: https://arxiv.org/abs/2604.15306
- 核心发现：LLM 在最短路径问题上空间泛化强，但长度 scaling 失败（递归不稳定）。数据覆盖设限，RL 提升稳定性但不扩限，推理时 scaling 无法拯救长度泛化失败。
- 对你最有价值的地方：解释了为什么你的 AI Agent 在处理更复杂任务时会失效——不是数据不够，而是递归推理不稳定。这是你做"个性化学习 Agent"时要注意的核心问题。

**② OpenMobile: Building Open Mobile Agents**
- ID: arXiv:2604.15093
- 链接: https://arxiv.org/abs/2604.15093
- 核心发现：开源移动 Agent 框架，用 Qwen2.5-VL 微调后达 51.7%，Qwen3-VL 达 64.7%（AndroidWorld）。核心是错误恢复数据的采集策略。
- 对你最有价值的地方：这是目前最完整的开源 Mobile Agent 方案，如果你想做"学习陪伴 Agent"，理解这套框架的思路（学习者+专家模型交替）可以直接用到你的产品设计里。

**③ Discovering Novel LLM Experts via Task-Capability Coevolution (AC/DC)**
- ID: arXiv:2604.14969
- 链接: https://arxiv.org/abs/2604.14969
- 顶会：ICLR 2026
- 核心发现：模型与任务共演化，单次训练即可发现比大模型更强的小模型，且无需 benchmark 优化。揭示了一种全新范式：模型自我驱动地探索能力空间。
- 对你最有价值的地方：这个思想和你想做的"个性化学习 Agent"高度相关——学生和 AI 学习内容共演化，AI 不断适应学生。

**④ Policy-Guided Hybrid Simulation (PGHS)**
- ID: arXiv:2604.15190（美团，SIGIR 2026 Industry）
- 链接: https://arxiv.org/abs/2604.15190
- 核心：LLM 推理分支 + ML 拟合分支双融合，模拟用户行为。美团实测误差 8.80%，比最强 baseline 提升 45.8%
- 对你最有价值的地方：LLM + 传统 ML 混合的实用范式。如果你的产品要做"学生学习行为预测"，这套架构可以直接参考。

**⑤ COEVO: Correctness + PPA Co-optimization in RTL Generation**
- ID: arXiv:2604.15001
- 链接: https://arxiv.org/abs/2604.15001
- 核心：代码生成中同时优化功能正确性和 PPA（功耗/性能/面积），用 Pareto 最优化替代单一目标
- 对你最有价值的地方：如果你的 Agent 要生成代码或练习题，这套多目标优化的思路可以借鉴
