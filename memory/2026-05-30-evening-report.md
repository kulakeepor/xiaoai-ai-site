### Paper 1（★★★★★）：RedundancyBench — Agent 轨迹里的冗余步骤，检测准确率仅 24.88%

**arXiv:2605.29893 | Minyang Hu et al. | cs.AI | 2026-05-28**

**为什么选这篇给老大细读：**

这是本周最直接关系到学习 Agent 效率的论文。它揭示了一个根本性问题：**即使是最好的方法，在检测 Agent 轨迹中冗余步骤的任务上准确率也只有 24.88%**——比随机猜测好不了多少。这意味着学习 Agent 在执行任务时，很可能充满了"看起来对但实际无用"的步骤，而现有的评估方法根本无法发现。

**核心发现：**

① **问题定义**：RedundancyBench 提出了一个新研究问题——检测 Agent 轨迹中的冗余步骤。每个步骤被标注为"对任务完成有贡献"或"冗余"。

② **关键数据**：即使是最先进的检测方法，准确率也只有 24.88%。这说明：
- Agent 的行为有大量隐含的冗余步骤
- 这些冗余步骤消耗资源但不贡献最终结果
- 现有的任务成功率指标完全忽略了这个问题

③ **典型失败案例**：多步推理中，Agent 可能在不同位置重复"验证已有信息"或"执行相似的子任务"。这些步骤通过 oracle 测试（任务成功），但实际上可以合并或删除。

**→ 对老大学习 Agent 的直接价值：**

① **"任务完成"≠"高效执行"**：学习 Agent 可以在表面上完成学习任务（生成正确答案），但过程中可能包含大量冗余思考步骤。如果老大想让学习 Agent 真正"学会"而非"表演学会"，需要评估的是推理链的效率，而非答案的正确率。

② **冗余检测 = 自我改进的基础**：如果 Agent 能检测出自己的冗余步骤，它就能自我修正。学习 Agent 的"反思"能力可以借鉴 RedundancyBench 的思路——不只是问"答案对不对"，而是问"这一步是否真正贡献了新信息"。

③ **资源消耗的隐形成本**：RedundancyBench 显示 Agent 轨迹中的冗余步骤消耗" substantial resources"。对于需要长时间运行的学习 Agent（比如规划一周的学习路径），冗余步骤会大幅增加推理成本。

**关键数字**：
- 最佳检测准确率：24.88%（低于随机）
- 任务覆盖：多种复杂任务，标注详细轨迹
- 适用场景：需要评估 Agent 执行效率的所有场景

**一句话总结**：RedundancyBench 揭示了 Agent 评估的核心盲区——任务成功不等于高效执行。学习 Agent 如果只评估"答案对不对"，会遗漏掉大量可优化的冗余推理步骤。引入"步骤贡献度"评估是下一代学习 Agent 的必修课。

### Paper 2（★★★★）：Moment-KV — 动量驱动的时间注意力聚合，KV 缓存压缩新范式

**arXiv:2605.29873 | Soumyadeep Jana et al. | cs.AI | 2026-05-28**

**为什么选这篇：**

这篇论文解决的是 LLM 部署中的一个核心工程问题——长文本生成的 KV 缓存瓶颈。对于学习 Agent 来说，长上下文（比如整个学期课程的上下文）是常见场景，KV 缓存的效率直接影响 Agent 的响应速度和成本。

**核心贡献：**

① **问题定位**：现有方法对 Prefill Cache（包含关键上下文）和 Decoding Cache（生成新 token）采用统一压缩，但 Prefill 压缩会损害性能（关键上下文被破坏）。Decoding Cache 的压缩才是真正未被充分探索的领域。

② **关键洞察——注意力有强时间模式**：
- 关键 token 获得跨长时域的持续注意力
- 局部推理涉及短时爆发性注意力
- 静态启发式方法（固定窗口）无法捕捉这种动态行为

③ **解决方案——Moment-KV**：
用**动量驱动的时间注意力聚合**来建模 token 重要性。核心思路：
- 注意力不是即时快照，而是持续累积的状态
- 随时间衰减的注意力聚合，同时捕捉长期影响和近期相关性
- 关键 token 在远期仍被保留；局部推理 burst 结束后被及时清理

**关键数字**：
- 长生成任务保真度提升：2.3-3.2%
- 解码延迟：保持不增加

**→ 对老大学习 Agent 的直接价值：**

学习 Agent 处理长学习路径（如"一学期微积分"）时：
1. **成本控制**：长上下文 = 大量 KV cache = 高成本。Moment-KV 的压缩思路可以直接应用于降低学习 Agent 的推理成本。
2. **长程依赖保留**：关键概念（核心定理）在整个学习路径中需要被持续保留（对应"长期注意力"），而中间步骤可以被及时清理。
3. **架构迁移**：Moment-KV 的"动量聚合"思路可以借鉴到学习 Agent 的记忆系统设计——不是简单的 LRU 替换，而是根据"对学习目标的重要度"动态调整记忆保留。

**一句话总结**：Moment-KV 揭示了 KV 缓存压缩的关键——注意力有动态时间模式，需要动量驱动的动态保留策略。学习 Agent 处理长学习路径时，借鉴此思路可以同时降低成本并保留关键学习记忆。
