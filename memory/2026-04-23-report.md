### Paper 1：RoboGrid — 评估 LLMs 作为 Context-Free Grammar Interpreters
**arXiv:2604.20811 | cs.AI | Hanqi Li**

**为什么重要**：当 LLM 被集成到 agentic 系统时，需要遵守机器可解释的动态定义接口。CFG（context-free grammar）是一种形式化表达这种接口的方式——给定一个 novel CFG，LLM 能否生成既**句法正确**又**行为功能正确**又**语义一致**的输出？

**核心发现（hierarchical degradation）**：
- **表层句法**能保持（surface syntax 没问题）
- **结构语义**在递归深度↑、分支复杂度↑时系统性崩溃
- CoT reasoning 能部分缓解，但语义对齐在极端深度**彻底消失**
- **"Alien"词表揭示**：LLM 依赖关键词的语义 boot-strapping，而非纯符号归纳——这意味着它们不是真正的符号推导器

**三维度 stress-test**：
1. 递归深度（从浅到深）
2. 表达式复杂度（从简单到嵌套）
3. 表面风格（从自然语言词汇到随机符号）

**对学习的启示**：
1. **LLM Agent 的可靠性边界**：如果你的工作流依赖 LLM 遵守动态定义的 grammar/规则，RoboGrid 告诉你这并不可靠——尤其是当规则有深层嵌套结构时
2. **Agent 设计建议**：对于需要强结构化输出的场景，考虑混合范式——LLM 负责语义层面，形式化验证层（symbolic checker）负责句法层面
3. **CoT 的局限**：Chain-of-Thought 能帮表层推理，但无法修复深层结构推理的不稳定性。这对 tool-use agent、code generation agent 的设计有直接启发

**一句话总结**：LLM 能模仿形式语言的表层，但不具备可靠的符号结构推理能力——在需要严格遵守 grammar 的 agentic 场景中，必须加 formal verification 层。

### Paper 2：Stream-CQSA — 通过灵活工作负载调度避免 Attention 计算中的 OOM
**arXiv:2604.20819 | cs.LG | Yiming Bian**

**为什么重要**：长上下文 LLM 的核心瓶颈是 exact self-attention 的二次方内存复杂度。现有方法（如 FlashAttention、PagedAttention）改善了效率，但**假设** QKV 整个 tensor 必须放进显存。Stream-CQSA 打破了这个假设。

**核心方法：CQS Divide + Stream-CQSA**：
- **CQS（Cyclic Quorum Sets）理论**：将 attention 分解为一组独立的子序列计算
- 每个子序列的 recomposition（重新组合）等于完整序列的 attention 结果——**数学上等价，无近似误差**
- **Stream-CQSA 调度框架**：根据可用显存预算，将 attention 切分为能放进显存的子问题
- 无需设备间通信（no inter-device communication）
- 单 GPU 可以 streaming 处理 billion-token 序列

**实验结果亮点**：
- 可预测的内存缩放（predictable memory scaling）
- 单 GPU 处理 billion-token 序列（10 亿 token）
- 完全保持 exact attention 的数学定义，无任何近似

**对学习的启示**：
1. **长上下文工程的范式转变**：从"如何让模型支持更长上下文"到"如何将 attention 计算本身变成可调度任务"——这是一个更基础的计算模型层面的改变
2. **硬件约束下的算法设计**：Stream-CQSA 的创新不在于新算法，而在于**去掉不合理的假设**（"QKV 必须整体放显存"）。这种"质疑隐含假设"的思维方式值得学习
3. **对部署的影响**：如果 exact attention 可以在单 GPU 上流式处理，那么现在需要多卡并行或近似方法的长上下文场景，都可能有更简单的解决方案

**一句话总结**：Stream-CQSA 用计算换内存——通过将 attention 分解为可独立调度的小问题，让 exact attention 在任意显存预算下单 GPU 就能跑，billion-token 上下文不再是工程难题。
