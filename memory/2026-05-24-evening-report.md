### Paper 1（★★★）：Mem-π — 自适应记忆：按需生成而非按需检索

**arXiv:2605.21463 | Xiaoqiang Wang et al. | cs.CL / cs.AI | 2026-05-20**

**为什么选这篇给老大细读：**

这篇论文解决的是 Agent 记忆系统的核心问题——**"检索式记忆"在复杂任务中的局限性**，直接对应老大学习 Agent 的记忆层设计。

**背景问题**：

现有记忆增强 Agent 主要靠"相似度检索"（similarity-based retrieval）从记忆库中拉取静态条目。但这种方法的本质缺陷是：
- 记忆库是静态的条目，不一定和当前任务语境对齐
- 相似度检索依赖表面语义匹配，容易拉偏
- 当任务复杂度增加时，检索命中率急剧下降

**核心贡献：按需生成（On-Demand Generation）**

Mem-π 的核心洞察是：**记忆不一定是被动的"存储→检索"，也可以是主动的"按需生成"**。

架构上，Mem-π 使用一个独立的小模型（专门的记忆生成器）来处理当前 Agent 的上下文，联合决定：
1. **何时生成**：当前任务是否需要记忆 guidance？（可以主动选择不生成）
2. **生成什么**：具体的 guidance 内容是什么？

训练方法上，使用决策-内容解耦的 RL 目标——模型可以"选择不生成"（abstain），只有当生成有帮助时才输出 concise、useful 的 guidance。这解决了传统方法"强制生成但容易误导"的根本问题。

**关键数据**：
- 在网页导航任务上比检索式 baseline 提升 **30%+ 相对改善**
- 跨 Agent 任务类型（网页导航 / 终端工具使用 / 文字具身交互）全面超越检索式和之前 RL 优化的记忆方法

**→ 对老大学习 Agent 的直接价值**：

老大正在做的学习 Agent 肯定需要记忆系统。Mem-π 提供了一种比"向量数据库检索"更优雅的思路：

1. **用专门的小模型生成学习建议**，而不是从记忆库检索固定的知识点
2. **记忆生成器自己决定何时需要生成 guidance**——学习 Agent 不需要每次都"查记忆"，而是只在需要时让记忆系统主动参与
3. **可以主动选择不生成**——这避免了"强制检索导致误导"的问题，记忆系统只在有用时才出现

**关键区别**：

| 方案 | 记忆内容 | 决策方式 |
|---|---|---|
| 检索式记忆 | 静态条目 | 相似度匹配（被动） |
| Mem-π | 按需生成的动态 guidance | RL 训练生成器自主决定何时+生成什么（主动） |

### Paper 2（★★★）：SR²AM — Agent 推理的三系统架构：什么时候"想"比"怎么想"更重要

**arXiv:2605.22138 | Mingkai Deng et al. | cs.AI / cs.CL / cs.LG | 2026-05-21**

**为什么选这篇给老大细读：**

这篇论文的核心贡献是回答了一个极其基础但从未被系统回答的问题：**Agent 什么时候应该"规划"，什么时候应该"直接反应"？**

这和老大学习 Agent 的核心挑战直接相关——学习 Agent 也需要决定："这道题我是简单查一下，还是要深度思考？"

**背景问题**：

主流方法是 Chain-of-Thought（CoT）——让模型在回答前展开推理过程。但 CoT 的问题是：**它假设所有问题都需要规划，但实际上很多简单问题不需要**。强制 CoT 会导致：
- Token 消耗大幅增加（效率低）
- 不一定带来准确率提升（有些问题想多了反而错）

**核心贡献：三系统架构（Three-System Architecture）**

SR²AM 将 Agent 决策分解为三个系统：

- **System II（Simulative Reasoning）**：基于世界模型的仿真推理——在决策前预测未来状态，类似于"先在脑子里推演一遍"
- **System III（Self-Regulation）**：自调节——学习得来的配置器，决定何时以及多深地调用规划（System II）
- **System I（Reactive Execution）**：反应式执行——处理细粒度动作，不需要规划

关键是 **System III 是一个学出来的"调度器"**——不是靠规则决定何时调用规划，而是通过 RL 训练学到的。它决定：**当前任务是否需要规划？需要规划多深？**

**关键数据（震撼）**：
- v0.1-8B 和 v1.0-30B 在数学/科学/表格分析/网页信息搜索任务上，Pass@1 性能与 **120-355B 和 685B-1T 参数的系统相当**
- v1.0-30B 比同类 Agent LLM 少用 **25.8-95.3% 的推理 token**
- RL 训练后规划 horizon 增加 22.8%，但规划频率仅增加 2.0%——说明 RL 学到的是"**更深度规划**"而非"**更频繁规划**"

**→ 对老大学习 Agent 的直接价值**：

1. **自适应学习深度**：学习 Agent 遇到简单问题（查个定义）→ 直接反应（System I）；遇到复杂问题（证明一个定理）→ 深度规划（System II + III）。这是 SR²AM 教给我们的。

2. **System III = 学习评估器**：Self-Regulation 模块本质上在做的事是"评估当前任务是否需要深度思考"。这正是老大学习 Agent 需要的功能——**评估学生的任务复杂度，决定学习策略的深度**。

3. **Token 效率的直接启示**：SR²AM 8B 模型靠自适应规划达到 685B-1T 模型的准确率，说明**规划策略比模型规模更重要**。学习 Agent 的"规划调度"能力，可能比换更大的模型更有价值。
