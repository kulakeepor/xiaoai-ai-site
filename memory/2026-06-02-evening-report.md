## 三、本周 arXiv 重点论文解读（向老大推荐 2 篇）

### 📖 论文 1：MAGIC — 训练-free 多模态 Coreset 选择
**arXiv:2605.26004 | Shristi Das Biswas et al. | ★★★★★（本周最重要）**

**核心问题**：LVLM 的指令微调数据集存在三个问题——冗余高、视觉依赖低、多模态推理行为覆盖严重不均。均匀子采样或基于分数的筛选都会得到次优子集。

**MAGIC 核心机制**（三个信号）：
1. **Multimodal Gain** — 衡量有视觉输入相比无视觉输入的似然提升
2. **Bridging Relevance** — 答案 token 在视觉 token 上的 grounding 清晰度
3. **Skill-Neuron Signatures** — 每个样本通过 top-activated 神经元表征功能计算

**三阶段流程**：低增益过滤 → 归一化质量排序 → 分桶预算分配（保留潜在多模态技能覆盖）

**效果**：在 LLaVA-665K 和 Vision-Flan 数据集上，20% 预算达到 100.3%（LLaVA）和 101.6%（Vision-Flan）相对全量微调性能；wall-clock 加速 73.7%。

**一句话**：不用训练、不用辅助选择器、不用聚类，只用 VLM 内在信号就能选出 20% 的数据达到 100%+ 效果。

**对老大的价值**：
- 学习 Agent 需要精选训练数据时，MAGIC 的 coreset 思路可以借鉴——不是越多越好，而是选得对
- Skill-Neuron Signatures 概念对理解"哪些样本教给模型什么技能"有启发

### 📖 论文 2：RAG-Match — 知识注入+层次推理+偏好校准
**arXiv:2605.25486 | Hengjun Jiang | ★★★★**

**核心问题**：搜索相关性判断在知识密集场景尤其困难——需要语义匹配+背景 grounded+多步推理+校准决策边界。现有模型依赖直接标签监督或浅层语义相似，无法处理隐式意图、事实等价、细粒度相关性区分。

**RAG-Match 三阶段框架**：
1. **知识增强预训练** — 强化 query-centered 语义 grounded
2. **层次推理对齐** — 将模型与结构化相关性推理对齐
3. **偏好决策校准** — 修正困难边界案例的决策层不一致

**效果**：在真实搜索相关性基准上全面超越 LLM-based 基线。

**一句话**：把 RAG 的检索相关性从"语义匹配"升级为"知识 grounded + 推理链 + 偏好优化"三层。

**对老大的价值**：
- 你的学习 Agent 如果涉及外部知识检索，RAG-Match 的层次推理对齐思路能显著提升答案质量
- 第三阶段的"偏好校准"对生成内容的可靠性很重要
